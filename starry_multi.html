Introduction¶

This page documents the starry.multi API, which is coded in C++ with a pybind11 Python interface. This API is identical in nearly all respects to the starry API, except that all internal computations are done using multi-precision floating point arithmetic. By default, starry.multi performs calculations using 32 digits, which roughly corresponds to 128-bit (quadruple) precision.

Note

The STARRY_NMULTI compiler flag determines the number of significant digits to use in multi-precision calculations and can be changed by setting an environment variable of the same name prior to compiling starry. See Installation for more information.

The Map classes¶

class starry.multi.Map(lmax=2)¶

Instantiate a starry surface map. Maps instantiated in this fashion are orthonormalized, so the total integrated luminosity of the map is \(2\sqrt{\pi} Y_{0,0}\).

Parameters:lmax (int) – Largest spherical harmonic degree in the surface map. Default 2.
evaluate(axis=(0, 1, 0), theta=0, x=0, y=0)¶

Return the specific intensity at a point (x, y) on the map. Users may optionally provide a rotation state. Note that this does not rotate the base map.

Parameters:
  • axis (ndarray) – Normalized unit vector specifying the body’s axis of rotation. Default \(\hat{y} = (0, 1, 0)\).
  • theta (float or ndarray) – Angle of rotation in degrees. Default 0.
  • x (float or ndarray) – Position scalar, vector, or matrix.
  • y (float or ndarray) – Position scalar, vector, or matrix.
Returns:

The specific intensity at (x, y).
rotate(axis=(0, 1, 0), theta=0)¶

Rotate the base map an angle theta about axis. This performs a permanent rotation to the base map. Subsequent rotations and calculations will be performed relative to this rotational state.

Parameters:
  • axis (ndarray) – Normalized unit vector specifying the body’s axis of rotation. Default \(\hat{y} = (0, 1, 0)\).
  • theta (float or ndarray) – Angle of rotation in degrees. Default 0.
flux(axis=(0, 1, 0), theta=0, xo=0, yo=0, ro=0)¶

Return the total flux received by the observer. Computes the total flux received by the observer from the map during or outside of an occultation.

Parameters:
  • axis (ndarray) – Normalized unit vector specifying the body’s axis of rotation. Default \(\hat{y} = (0, 1, 0)\).
  • theta (float or ndarray) – Angle of rotation. Default 0.
  • xo (float or ndarray) – The x position of the occultor (if any). Default 0.
  • yo (float or ndarray) – The y position of the occultor (if any). Default 0.
  • ro (float) – The radius of the occultor in units of this body’s radius. Default 0 (no occultation).
Returns:

The flux received by the observer (a scalar or a vector).
get_coeff(l, m)¶

Return the (l, m) coefficient of the map.

Note

Users can also retrieve a coefficient by accessing the [l, m] index of the map as if it were a 2D array. Single slice indexing is also allowed.

Parameters:
  • l (int) – The spherical harmonic degree, ranging from 0 to lmax.
  • m (int) – The spherical harmonic order, ranging from -l to l.
set_coeff(l, m, coeff)¶

Set the (l, m) coefficient of the map.

Note

Users can also set a coefficient by setting the [l, m] index of the map as if it were a 2D array. Single slice indexing is also allowed.

Parameters:
  • l (int) – The spherical harmonic degree, ranging from 0 to lmax.
  • m (int) – The spherical harmonic order, ranging from -l to l.
  • coeff (float) – The value of the coefficient.
reset()¶

Set all of the map coefficients to zero.

The highest spherical harmonic order of the map. Read-only.

y¶

The spherical harmonic map vector. Read-only.

p¶

The polynomial map vector. Read-only.

g¶

The Green’s polynomial map vector. Read-only.

s¶

The current solution vector r. Read-only.

minimum()¶

Find the global minimum of the map. This routine wraps scipy.optimize.minimize to find the global minimum of the surface map. This is useful for ensuring that the surface map is nonnegative everywhere.

Note

Because this routine wraps a Python wrapper of a C function to perform a non-linear optimization in three dimensions, it is slow and should probably not be used repeatedly when fitting a map to data!

add_gaussian()¶

Add the spherical harmonic expansion of a gaussian to the current map. This routine adds a gaussian-like feature to the surface map by computing the spherical harmonic expansion of a 3D gaussian constrained to the surface of the sphere. This is useful for, say, modeling star spots or other discrete, localized features on a body’s surface.

Note

Because this routine wraps a Python function, it is slow and should probably not be used repeatedly when fitting a map to data!

Parameters:
  • sigma (float) – The standard deviation of the gaussian. Default 0.1
  • amp (float) – The amplitude. Default 1.0, resulting in a gaussian whose integral over the sphere is unity.
  • lat (float) – The latitude of the center of the gaussian in degrees. Default 0.
  • lon (float) – The longitude of the center of the gaussian in degrees. Default 0.
load_array(image)¶

Load a lat-lon image array. This routine loads a 2D numpy array, computes its spherical harmonic expansion up to degree lmax, and sets the map vector.

Parameters:image (ndarray) – The 2D numpy lat-lon array.
load_image(image)¶

Load an image from file. This routine loads an image file, computes its spherical harmonic expansion up to degree lmax, and sets the map vector.

Parameters:image (str) – The full path to the image file.
load_healpix(image)¶

Load a healpix image array. This routine loads a healpix array, computes its spherical harmonic expansion up to degree lmax, and sets the map vector.

Parameters:image (ndarray) – The ring-ordered healpix array.

class starry.multi.LimbDarkenedMap(lmax=2)¶

Instantiate a starry limb-darkened surface map. This differs from the base Map class in that maps instantiated this way are radially symmetric: only the radial (m = 0) coefficients of the map are available. Users edit the map by directly specifying the polynomial limb darkening coefficients u, starting with \(u_1\) (linear limb darkening). The coefficient \(u_0\) is fixed to enforce the correct normalization.

Warning

Unlike Map, maps instantiated this way are normalized so that the integral of the specific intensity over the visible disk is unity. This is convenient for using this map to model stars: the unocculted flux from the star is equal to one, regardless of the limb-darkening coefficients!

Parameters:lmax (int) – Largest spherical harmonic degree in the surface map. Default 2.

Parameters:
  • x (float or ndarray) – Position scalar, vector, or matrix.
  • y (float or ndarray) – Position scalar, vector, or matrix.
Returns:

The specific intensity at (x, y).

roots()¶

Count the number of roots of the specific intensity using Sturm’s theorem. Physical maps with positive semi-definite specific intensity should have zero roots. If this number is nonzero, your star has negative brightness somewhere!

class starry.multi.Planet(lmax=2, r=0.1, L=0, axis=(0, 1, 0), prot=0, a=50, porb=1, inc=90, ecc=0, w=90, Omega=0, lambda0=90, tref=0)¶

Instantiate a planetary Body object. Instantiate a planet. At present, starry computes orbits with a simple Keplerian solver, so the planet is assumed to be massless.

  • ecc (float) – Orbital eccentricity. Default 0.

  • Omega (float) – Longitude of ascending node in degrees. Default 0.
  • lambda0 (float) – Mean longitude at time tref in degrees. Default 90.

class starry.multi.System(bodies, scale=0, exposure_time=0, exposure_tol=1e-8, exposure_max_depth=4)¶

Instantiate an orbital system.

Parameters:
  • bodies (list) – List of bodies in the system, with the primary (usually the star) listed first.
  • scale (float) – This parameter sets the lengthscale for computing the light travel time delay and is simply equal to the radius of the star in \(R_\odot\). Default 0, meaning the light travel time effect is not computed.
  • exposure_time (float) – The exposure time of the observations in days. If nonzero, the flux will be integrated over this exposure time. Note that setting this will result in slower run times, since the integrated flux is computed numerically. Default 0.
  • exposure_tol (float) – Tolerance of the recursive method for integrating the flux over the exposure time. Default 1e-8.
  • exposure_maxdepth (int) – Maximum recursion depth for the exposure calculation. Default 4.
compute(time)¶

Compute the system light curve analytically. Compute the full system light curve at the times given by the time array and store the result in flux. The light curve for each body in the system is stored in the body’s flux attribute.

Parameters:time (ndarray) – Time array, measured in days.
flux¶

The computed system light curve. Must run compute() first. Read-only.

exposure_time¶

The exposure time of the observations in days. If nonzero, the flux will be integrated over this exposure time.

exposure_tol¶

Tolerance of the recursive method for integrating the flux over the exposure time.

exposure_max_depth¶

Maximum recursion depth for the exposure calculation.
